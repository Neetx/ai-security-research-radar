- 2026-06-30 — [AI-Infra-Guard: A Unified Framework for Multi-Layer Agent Red Teaming](https://arxiv.org/abs/2606.31227) — open-source framework that matches a detection paradigm to each agent layer: deterministic rule matching over 75+ AI components / 1,400+ vuln rules, LLM-driven agentic auditing of MCP servers and agent-skill packages, multi-turn black-box agent red teaming, and a jailbreak harness (26+ operators, 16 datasets); claims to be the only open-source tool spanning infra→protocol/tool→behavior→model incl. skill supply-chain auditing — a practical red-team toolkit for the whole agent stack.
- 2026-06-27 — [Capability Gates Are Not Authorization: Confused-Deputy Failures in LLM Agent Frameworks](https://arxiv.org/abs/2606.28679) — concrete audit showing LangChain/LangGraph, LlamaIndex and the Stripe Agent Toolkit all gate tool exposure but none re-authorizes each model-emitted call against its concrete arguments before execution; ships ScopeGate (a fail-closed PDP/PEP: scope → authorization → money ceiling → idempotency → default-deny). The practical reference for why "the agent has the tool" is not "the agent may use it on these arguments," and a deployable pattern to fix it.
- 2026-06-25 — [On the Inseparability of Instructions and Data in Shared-Embedding Sequence Models](https://arxiv.org/abs/2606.27567) — proves that in shared-embedding architectures lacking enforced control/data separation, PERFECT prompt-injection prevention is mathematically impossible: formalizes "Semantic-Faithful Control" and shows it is unachievable via provenance-recovery impossibility, control-path exposure through shared attention value-aggregation, and a finite-coverage invariance gap — grounded in measurements on production tokenizers/models. The theoretical why behind "every prompt-injection defense proposed so far has been broken," and the case for out-of-band/architectural separation (cf. the OOB-defense SoK below).
- 2026-06-25 — [Adaptive Evaluation of Out-of-Band Defenses Against Prompt Injection in LLM Agents](https://arxiv.org/abs/2606.26479) — organizes the 2024–2026 wave of out-of-band agent defenses (CaMeL, FIDES, Progent, RTBAS, FORGE) as instances of classical Biba integrity / reference-monitor / least-privilege, then stress-tests them with adaptive attacks — the reference map for "enforce security outside the model" agent defenses and how durable their near-100% AgentDojo numbers really are.
- 2026-06-24 — [SoK: AI Secure Code Generation (Patir, Guo, Cai, Hu)](https://arxiv.org/abs/2606.25195) — systematizes what today's models and coding agents can and cannot do for secure code via a three-level framework (natural-language understanding of secure-coding principles → secure generation → agentic workflows), mapping where prompting / fine-tuning / RL / agentic methods actually help and why substantial security failures persist — the reference map for the "(in)security of AI-written code" problem.
